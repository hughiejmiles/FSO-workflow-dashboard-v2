Stage 1: Scoping & Concept Development

Preliminary decision point - determines External vs Internal path

Project Memory

Custom Instructions

Project Description

Core job: Determine IF and HOW to proceed with training product development

What comes in:

  • Trigger event (legislation, industry feedback, framework update)
  • Initial stakeholder input
  • Enrolment data (for updates)
  • Regulatory context
  • Strategic priorities from DEWR or FSO Leadership

What goes out:

  • Decision: Proceed (External/Internal) or Don't Proceed
  • Preliminary scope document
  • Risk assessment with mitigation strategies
  • Resource plan and timeline
  • Stakeholder engagement strategy

Breaks downstream if:

  • Wrong path chosen (should be External but goes Internal, or vice versa)
  • Insufficient evidence gathered to support decisions
  • Scope too broad or too narrow for allocated resources
  • Stakeholder misalignment from start
  • Framework updates not monitored (proceeding with outdated frameworks)

Fixed Skills Required (Universal - All Projects)

  • TPOF requirements and compliance framework
  • AQF Handbook (Levels 2-5) - understanding of level descriptors
  • Training Package Products Policy
  • RTO Standards 2025 - assessment and delivery requirements
  • Assessment Principles and Rules of Evidence
  • Australian Government Style Manual - Plain English and accessibility
  • NCVER enrolment data interpretation
  • DEWR process requirements and milestone frameworks

Variable Skills Required (Project-Specific)

  • Sector-specific frameworks (DigComp 3.0, ASFI Capability Framework, ACMA standards)
  • Regulatory landscape understanding (ASIC/APRA for finance, ACMA for telco)
  • International framework comparison and adaptation
  • Technology evolution tracking in relevant sector
  • Workforce projections and labour market analysis

Universal Context Documents (Fixed)

  • TPOF template requirements and structure specifications
  • AQF Level Descriptors (Levels 2-5) - detailed characteristics
  • Training Package Products Policy (current version)
  • Standards for RTOs 2025
  • Assessment Principles (Validity, Reliability, Fairness, Flexibility)
  • Rules of Evidence (Sufficiency, Authenticity, Currency)
  • NCVER Statistics access and interpretation guides
  • DEWR Activity Proposal templates and submission requirements

Variable Context Documents (Project-Specific)

  • Current legislation/regulatory documents (sector-specific - e.g., Treasury Laws Amendment Act, ACMA Cabling Provider Rules)
  • Industry reports and workforce projections
  • Job task analysis and role descriptions
  • Professional body requirements (CPA Australia, CA ANZ, CFA Institute, etc.)
  • International framework comparisons (Singapore, Ireland, UK approaches)
  • Existing training product analysis (current units, qualifications, skill sets)
  • Superseded unit lists and mapping
  • Overlap analysis (e.g., UEE/ICT unit duplication)
  • Previous consultation feedback and validation records
  • AISC deleted units review (for potential reinstatement)

Quality Criteria

Clear evidence-based rationale (not just "someone asked for it")
Strategic framework thinking (fits broader system, not isolated)
Stakeholder alignment secured early
Realistic resource/timeline planning
Risk assessment with mitigation strategies
Clear path decision with justification
Compliance mapping upfront
Framework currency verified (not using outdated versions)

Stage 2A: External Scoping

Industry-led workshops, writing team supports

Project Memory

Custom Instructions

Project Description

Core job: Support industry expert to scope training product updates/development

What comes in:

  • Initial scope from Stage 1
  • Current training products (units, qualifications, skill sets)
  • Enrolment/usage data (NCVER Statistics 2019-2024)
  • Regulatory/compliance documentation
  • Stakeholder list and consultation framework
  • Historical consultation feedback

What goes out:

  • Comprehensive scoping report from external consultant
  • Categorised recommendations (Retain/Delete/Update/Further Consultation with action codes)
  • Evidence base for all decisions
  • Supporting documentation (e.g., 14 attachments for ICT Telco: Qual/SS/UoC Summary Matrix, Feedback Log, Superseded UoCs, AISC Deleted Units, Regulatory Requirements, Learning Pathways, Access & Equity Coverage, Sample UoCs, Design Considerations, Trial Qualifications, Industry Materials)
  • Action plan with responsibility assignment (Integration Team vs External Writers vs Project Team)
  • Integration Writing Team implementation brief

Breaks downstream if:

  • Insufficient briefing to external consultant
  • Missing regulatory/framework documentation
  • Wrong SMEs consulted (general practitioners instead of specific capability experts)
  • Recommendations not actionable or unclear
  • Unclear responsibility assignment between teams
  • Contradictory feedback not resolved (requires "Further Consultation" category)

Fixed Skills Required (Universal - All Projects)

  • TPOF template requirements and structure
  • AQF level descriptors and expectations
  • Training Package compliance standards
  • FSO brand guidelines (Montserrat typography, colour palette, Australian English)
  • Assessment principles (validity, reliability, fairness, flexibility)
  • Stakeholder coordination and consultation protocols
  • Consultation log maintenance and synthesis

Variable Skills Required (Project-Specific)

  • Sector frameworks (ACMA standards for telco, ASFI for finance, DigComp for digital capability)
  • Regulatory compliance expertise (sector-specific - ACMA, ASIC, APRA)
  • Industry stakeholder landscape understanding
  • Technology currency assessment (what's current vs emerging vs outdated)
  • Cross-JSC coordination (e.g., UEE/ICT overlap requiring PSO consultation)

Universal Context Documents (Fixed)

  • TPOF requirements and template specifications
  • AQF Handbook (relevant levels)
  • Training Package Products Policy
  • Standards for RTOs 2025
  • Assessment Principles and Rules of Evidence
  • FSO brand guidelines and style requirements
  • Current training products (units, qualifications, skill sets for review)
  • Consultation log template and protocols

Variable Context Documents (Project-Specific)

  • Sector regulatory documents (e.g., ACMA Cabling Provider Rules 2025 + Explanatory Statement for ICT Telco)
  • Australian Standards (sector-specific - e.g., S009 ES1/ES2/ES3 for telco)
  • Industry reports and technology roadmaps (5G, IoT, AI, LEOS for telco; ISSB, TCFD, TNFD for finance)
  • Enrolment data analysis (NCVER Statistics showing zero enrolments, usage patterns)
  • Superseded unit lists and replacement mapping
  • AISC deleted units review (for potential reinstatement consideration)
  • Overlap analysis documentation (e.g., 19 UEE units overlapping with 18 ICT units)
  • Access and equity documentation and current coverage analysis
  • Learning pathway analysis and qualification packaging structures
  • Sample UoCs with wording analysis
  • Design and structural considerations (TPOF Reform implications)
  • Victorian trial qualifications or state-specific training products (where relevant)

Quality Criteria

External consultant properly briefed with comprehensive materials
All relevant stakeholders consulted (Industry, RTO, SME)
Clear categorisation framework applied (R/D/U/FC with rationale)
Evidence-backed recommendations (not just opinions)
Actionable implementation plan with specific steps
Responsibility assignment clear (Integration vs External Writers vs Project Team)
Supporting documentation comprehensive and organised
Contradictory feedback identified and resolution pathway clear

Stage 2B: Internal Scoping

Writing team-led, Technical Committee validates

Project Memory

Custom Instructions

Project Description

Core job: Translate industry/international framework into structured VET training product plan

What comes in:

  • Initial scope from Stage 1
  • Complete framework specifications (DigComp 3.0, ASFI Capability Framework, DEC AI Literacy, Cybersecurity Skills Framework)
  • VET compliance requirements (TPOF, AQF, Training Package Products Policy)
  • DEWR Activity Proposal approved with milestone framework
  • Strategic context (Purpose 1/2/3 qualification types, cross-sector needs, RPL requirements)
  • Previous framework versions (for understanding transition rationale)

What goes out:

  • Complete unit structure (all unit codes, titles, application descriptions)
  • Development matrices (showing all units across levels and competence areas)
  • Framework mapping logic (how source framework translates to VET units)
  • Phased development plan (e.g., Foundation → Intermediate → Advanced → Highly Specialised)
  • Quality assurance approach and criteria
  • Project Brief or Overview Document (comprehensive 18+ page presentation)
  • Risk management plan with mitigation strategies
  • Consolidation rationale documentation (explaining reduction from 80 elements to 39 units)

Breaks downstream if:

  • Framework translation logic unclear or arbitrary
  • AQF placement inappropriate for actual complexity
  • Unit structure doesn't support clear pathways
  • Quality criteria not defined before drafting begins
  • Development methodology not systematic (leads to inconsistency)
  • Framework updates occur mid-development without strategic pause
  • Multi-framework integration logic unclear (e.g., 80% DigComp + 20% AI + 20% Cybersecurity)

Fixed Skills Required (Universal - All Projects)

  • TPOF ASK template mastery - structure, requirements, compliance
  • AQF Level Descriptors deep knowledge (Levels 2-5) - autonomy, complexity, depth/breadth characteristics
  • Training Package Products Policy compliance
  • Assessment principles and Rules of Evidence deep understanding
  • Framework analysis and systematic translation capability
  • Systematic simplification ("lego to duplo" methodology)
  • Iterative refinement and quality evolution tracking
  • Change register documentation rigour

Variable Skills Required (Project-Specific)

  • Source framework deep expertise (DigComp 3.0 structure and proficiency levels, ASFI Capability Framework five core capabilities)
  • Multi-framework integration (when combining DigComp + AI Literacy + Cybersecurity, or ASFI + ISSB + TCFD + TNFD)
  • Cross-sector contextualisation (healthcare, construction, retail, hospitality, manufacturing, education for digital; banking, investment, insurance, accounting for finance)
  • RPL pathway design (vertical progressive units, flexible entry/exit points)
  • Attitudes integration (converting framework attitudes to observable VET Skills at higher levels)
  • First Nations co-design protocols (when relevant - Sustainable Finance example)

Universal Context Documents (Fixed)

  • TPOF ASK template requirements and detailed specifications
  • AQF Handbook (Levels 2-5) - comprehensive level descriptors
  • Training Package Products Policy (current version)
  • Standards for RTOs 2025
  • Assessment Principles (Validity, Reliability, Fairness, Flexibility)
  • Rules of Evidence (Sufficiency, Authenticity, Currency)
  • Australian Government Style Manual - Plain English and accessibility
  • FSO brand guidelines (Montserrat, colour palette, Australian English)
  • FSO glossary (when comprehensive version developed)
  • Change register template

Variable Context Documents (Project-Specific)

  • Complete framework specifications:
    • Digital projects: DigComp 3.0 complete specification (all elements, proficiency descriptors), DEC AI Literacy Framework (five dimensions), Cybersecurity Skills Framework
    • Finance projects: ASFI Capability Framework (complete), ISSB Standards (IFRS S1, S2), TCFD Recommendations, TNFD Framework
  • International framework comparisons (Singapore, Ireland, UK approaches for benchmarking)
  • DEWR Activity Proposal (approved scope and milestone requirements)
  • Purpose 1/2/3 qualification guidelines (design implications for different qualification types)
  • Cross-sector application examples (industry-specific contextualisation)
  • Target qualification packaging rules (which qualifications will include these units)
  • Previous framework versions (e.g., DigComp 2.2 for understanding transition to 3.0)
  • Regulatory documentation (sector-specific):
    • Finance: Treasury Laws Amendment Act, ASRS, ASIC guidance, APRA CPS 229
    • Digital: Qualification reform requirements, cross-sector VET standards
  • Framework comparison tables and mapping matrices
  • Consolidation ratio documentation (showing systematic reduction approach)

Quality Criteria

Clear evidence-based rationale for all structural decisions
Systematic framework translation (transparent, defensible methodology)
Comprehensive coverage without duplication
Appropriate AQF placement with justification
Logical pathway structure (clear progression, entry/exit flexibility)
Realistic timeline and resource allocation
Quality criteria defined upfront (not discovered during drafting)
Technical Committee validation secured early
Framework currency verified (not proceeding with outdated frameworks)
Multi-framework integration logic clear and documented

Stage 3A: External Drafting

Writing team guides external consultants

Project Memory

Custom Instructions

Project Description

Core job: Coordinate external technical experts to draft high-quality, compliant units

What comes in:

  • Scoping report with categorised units (which 125 units need updating)
  • Responsibility assignment (which writers handle which units based on technical expertise)
  • Template decision from Technical Committee (E&PC or ASK format)
  • All reference materials (regulatory documents, Australian Standards, frameworks)
  • Quality criteria and exemplar units (showing expected standard)
  • Feedback Log from scoping (stakeholder input by training product)
  • Superseded unit lists (what needs replacing with current equivalents)

What goes out:

  • Draft units (TPOF compliant structure)
  • Updated content including:
    • Current technology (5G, IoT, AI, LEOS, PoE, IPv6/SDN)
    • Regulatory compliance (ACMA Cabling Provider Rules 2025)
    • Current standards (AS/NZS 14763.2, AS 61935, Australian Standards S009)
    • Safety requirements (fire protection, confined spaces, working at heights - updated)
    • Workplace currency (actual industry practices, not outdated methods)
  • Documentation of changes made (what was updated and why)
  • Self-QA from external writers (initial quality checking)
  • Ready for Integration Writing Team QA (systematic review)

Breaks downstream if:

  • Insufficient briefing to external writers (expectations unclear, quality standards unknown)
  • Template/format confusion (E&PC vs ASK not clarified, results in wrong format)
  • Missing reference materials (regulatory docs not final, standards outdated)
  • Inconsistent quality across writers (some excellent, others requiring substantial rework)
  • Late-stage discovery of major issues (fundamental problems found at final submission)
  • Assessment Conditions unrealistic for RTOs (specifying equipment/resources not available)
  • UEE/ICT overlap unidentified (external writers don't know about 19 UEE units overlapping with 18 ICT units)
  • Regulatory misinterpretation (ACMA rules applied incorrectly, mandatory vs guidance confused)
  • Safety requirements incomplete (fire protection, confined spaces, working at heights partially addressed)

Fixed Skills Required (Universal - All Projects)

  • TPOF ASK template requirements (structure, sections, compliance rules)
  • AQF level descriptors and appropriate complexity expectations
  • Training Package Products Policy compliance
  • Assessment principles and Rules of Evidence (validity, reliability, fairness, flexibility, sufficiency, authenticity, currency)
  • Australian Government Style Manual compliance
  • FSO brand guidelines application (Montserrat typography, colour palette, Australian English spelling)
  • Quality assurance frameworks and systematic review processes
  • External writer coordination and feedback management
  • Version control and change tracking across multiple drafts

Variable Skills Required (Project-Specific)

  • Sector regulatory requirements deep knowledge (ACMA for telco, ASIC/APRA for finance)
  • Australian Standards expertise (sector-specific - e.g., S009 ES1/ES2/ES3 for telecommunications)
  • Technology currency assessment (what's current, emerging, or outdated in specific sector)
  • Safety-critical requirements understanding (where relevant - network integrity, worker safety, community protection)
  • Industry authenticity validation (workplace practices reflect actual industry operation)
  • Cross-JSC awareness (identifying overlaps like UEE/ICT units requiring coordination)

Universal Context Documents (Fixed)

  • TPOF template requirements and detailed specifications
  • AQF Handbook (relevant levels with detailed descriptors)
  • Training Package Products Policy (current version)
  • Standards for RTOs 2025
  • Assessment Principles (Validity, Reliability, Fairness, Flexibility)
  • Rules of Evidence (Sufficiency, Authenticity, Currency)
  • Australian Government Style Manual
  • FSO brand guidelines (typography, colours, formatting)
  • FSO glossary (standardised terminology)
  • Change register template (for tracking modifications)
  • External writer briefing pack template

Variable Context Documents (Project-Specific)

  • Scoping report with categorised units (R/D/U/FC action codes)
  • Feedback Log (stakeholder input organised by training product)
  • Superseded UoC list (units requiring replacement with current equivalents)
  • Regulatory documentation:
    • ICT Telco: ACMA Cabling Provider Rules 2025, Explanatory Statement, ACMA CPR Pathways to Cabler Registration
  • Australian Standards specifications (sector-specific):
    • ICT Telco: AS/NZS 14763.2 (structured cabling), AS 61935 (testing standards), Australian Standards S009 (ES1, ES2, ES3)
  • Safety requirements documentation:
    • Fire protection standards
    • Confined spaces regulations
    • Working at heights requirements
  • Learning pathway analysis (CITT Telco Product Learning Pathways Report)
  • Sample UoCs with wording analysis (exemplars showing quality expectations)
  • Design and Structural Considerations (TPOF Reform implications)
  • Victorian trial qualifications (22586VIC - when relevant for benchmarking)
  • Additional Industry Related Materials
  • Current ICT units (for consistency checking)
  • UEE units (for overlap awareness - 19 UEE units overlapping with 18 ICT units)
  • Technology roadmaps (5G, IoT, AI, LEOS, PoE, IPv6/SDN evolution)

Quality Criteria

All external writers properly briefed upfront (comprehensive materials, clear expectations)
Comprehensive reference materials provided (regulatory, standards, exemplars)
Regular check-ins conducted (not just final submission)
Consistent quality across writers (terminology aligned, standards maintained)
TPOF compliance verified (structure, sections, format correct)
AQF level appropriate (language and complexity match intended level)
Assessment validity confirmed (Performance Evidence observable/measurable, Knowledge Evidence appropriate depth)
Realistic conditions for RTOs (equipment/resources actually available, environments achievable)
No duplication or overlap (especially UEE/ICT awareness maintained)
Regulatory compliance verified (ACMA rules, Australian Standards correctly applied)
Safety requirements complete (fire protection, confined spaces, working at heights fully addressed)
Technology currency confirmed (current practices, not outdated methods)

Stage 3B: Internal Drafting

Writing team authors units

Project Memory

Custom Instructions

Project Description

Core job: Author complete, compliant units through systematic framework translation

What comes in:

  • Complete unit structure from Internal Scoping (all codes, titles, application descriptions)
  • Framework specifications (DigComp 3.0 complete, ASFI Capability Framework, DEC AI Literacy, Cybersecurity Skills Framework)
  • Development matrices and mapping logic (how framework elements consolidate)
  • Phased development plan (sequence and timeline)
  • Quality criteria established (what "good" looks like defined upfront)
  • Previous exemplar units (if available from earlier phases or other projects)
  • Regulatory documentation (sector-specific compliance requirements)

What goes out:

  • Complete drafted units (TPOF ASK template compliant):
    • Unit code and title
    • Application (descriptor of skills/knowledge and work contexts)
    • Application, Skills, Knowledge (ASK template sections)
    • Foundation Skills (explicit mapping)
    • Assessment Requirements (Performance Evidence, Knowledge Evidence, Assessment Conditions)
    • Unit Mapping Information (links to other units, equivalencies)
  • Framework mapping documentation (transparent audit trail)
  • Change registers (comprehensive modification tracking - what changed, why, when, impact)
  • Self-QA documentation (verification performed during drafting)
  • Coding system transparent (K1-K13, S1-S6, PE1-PE4 shows framework alignment)
  • Ready for QA & Compliance stage (systematic validation)

Breaks downstream if:

  • Framework alignment drift (loses connection to source framework, consolidation without rationale)
  • AQF level inappropriate (language says "AQF 4" but complexity is actually AQF 3, or vice versa)
  • Internal inconsistency (Application promises capability that ASK sections don't deliver)
  • Assessment validity issues (Performance Evidence not observable, Assessment Conditions unrealistic)
  • Cross-unit terminology inconsistency (same concept using different terms across units)
  • Vertical progression gaps (Foundation → Intermediate jump too big, or plateau with no advancement)
  • Multi-framework integration confusion (80% DigComp + 20% AI + 20% Cybersecurity overlap unclear)
  • Attitudes integration failure (framework attitudes not converted to observable Skills)
  • Working solo limitations (can't see own assumptions, terminology drift unnoticed, complexity creep undetected)

Fixed Skills Required (Universal - All Projects)

  • TPOF ASK template mastery (structure, requirements, section specifications)
  • AQF level descriptors deep knowledge (Levels 2-5 autonomy, complexity, depth/breadth characteristics)
  • Assessment design expertise (creating observable, measurable, authentic evidence requirements)
  • Framework analysis and systematic translation capability
  • Systematic simplification methodology ("lego to duplo" without losing functionality)
  • Iterative refinement capability (learning from each unit, applying improvements to next)
  • Self-QA rigour (compensating for lack of external validation during drafting)
  • Change register documentation discipline
  • Plain English writing (accessibility, conciseness, clarity)

Variable Skills Required (Project-Specific)

  • Source framework deep expertise:
    • Digital: DigComp 3.0 complete structure (5 competence areas, 4 proficiency levels, 80+ elements), proficiency descriptors (Basic, Intermediate, Advanced, Highly Advanced)
    • Finance: ASFI Capability Framework (5 core capabilities, 3 proficiency levels, Knowledge and Abilities structure)
  • Multi-framework integration expertise:
    • Digital: DigComp (80%) + DEC AI Literacy (20%) + Cybersecurity Skills (20%) with overlap logic
    • Finance: ASFI + ISSB + TCFD + TNFD coherent integration
  • Cross-sector contextualisation (providing diverse industry examples):
    • Digital: Healthcare, construction, retail, hospitality, manufacturing, education applications
    • Finance: Banking, investment management, insurance, accounting sector applications
  • Regulatory compliance deep knowledge (sector-specific):
    • Finance: Treasury Laws Amendment Act, ASRS, ASIC disclosure requirements, APRA CPS 229
  • Cultural competence:
    • First Nations co-design protocols (Sustainable Finance: FPIC principles, economic self-determination)
    • Accessibility and inclusive design principles
  • Attitudes integration expertise (converting framework attitudes to observable VET Skills at higher levels)
  • RPL pathway design (vertical progressive units, flexible entry/exit points)

Universal Context Documents (Fixed)

  • TPOF ASK template requirements (detailed specifications for each section)
  • AQF Handbook (Levels 2-5) - comprehensive level descriptors with examples
  • Training Package Products Policy (current version)
  • Standards for RTOs 2025
  • Assessment Principles (Validity, Reliability, Fairness, Flexibility)
  • Rules of Evidence (Sufficiency, Authenticity, Currency)
  • Australian Government Style Manual - Plain English, accessibility, inclusive language
  • FSO brand guidelines (Montserrat typography, colour palette #000000, #FF4200, #FFFFFF, #EAEAEA, #17396F, #EA9B3A, Australian English spelling)
  • FSO glossary (standardised terminology - e.g., "AQF Level" vs "proficiency level" distinction)
  • Change register template

Variable Context Documents (Project-Specific)

  • Complete framework specifications (full detail, not summaries):
    • Digital projects: DigComp 3.0 complete specification (all elements, proficiency descriptors), DEC AI Literacy Framework (five dimensions with progression), Cybersecurity Skills Framework (threat awareness, protection, response)
    • Finance projects: ASFI Capability Framework complete specification (five core capabilities: Strategy, Risk Management, Products, Reporting, Mindset), ISSB Standards (IFRS S1, S2), TCFD Recommendations, TNFD Framework
  • Regulatory documentation (sector-specific):
    • Finance: Treasury Laws Amendment Act (mandatory climate disclosure legislation), ASRS (Australian Sustainability Reporting Standards), ASIC guidance (disclosure obligations), APRA CPS 229 (climate financial risk requirements)
  • Australian Standards (sector-specific when relevant)
  • Target qualification packaging rules (which qualifications will include these units as electives):
    • Digital: BSB Training Package integration
    • Finance: FNS40222, FNS50222, FNS60222 Certificate IV/Diploma/Advanced Diploma in Accounting, BSB50920 Diploma of Quality Auditing
  • Previous exemplar units (from earlier phases - e.g., first 20 Digital Capability units inform next 19)
  • International comparison documentation (Singapore, Ireland, UK frameworks for benchmarking adaptation approaches)
  • Industry glossaries (sector-specific terminology):
    • Finance: Sustainable finance terms, ESG definitions, climate risk taxonomy
    • Digital: AI/ML terminology, cybersecurity concepts, digital platform types
  • Cultural validation protocols:
    • First Nations co-design requirements (Sustainable Finance: integrated units addressing economic self-determination)
  • Framework comparison tables (showing how elements map across international/industry frameworks and VET structure)
  • Consolidation ratio documentation (Digital: 80 DigComp elements → 39 units with systematic approach)
  • Multi-framework integration logic documentation (how 80% + 20% + 20% works, overlaps identified, gaps managed)
  • Cross-sector application examples (specific scenarios for diverse industries)
  • Previous framework versions (e.g., DigComp 2.2 for understanding transition rationale to 3.0)

Quality Criteria

Explicit, transparent framework mapping (every K/S statement traces to framework elements)
AQF level-appropriate language and complexity (verbs match descriptors, tasks appropriate)
Observable, measurable Performance Evidence (using "demonstrate," "apply," "implement" not "understand," "appreciate")
Realistic Assessment Conditions (resources RTOs can provide, environments achievable, timeframes reasonable)
Clear, accessible language (Plain English, jargon explained, straightforward sentences)
Internal consistency (Application descriptor aligns with ASK content delivered)
Cross-unit consistency (same concept = same term, FSO glossary applied, prerequisites logical)
Systematic quality (change registers maintained, Technical Committee feedback addressed, compliance verified)
Iterative excellence (visible learning, later units more refined, methodology documentation shows evolution)
Complete framework coverage (all elements addressed, no gaps)
No unintended duplication (consolidation rationale clear)
Strategic coherence (fits pathway, prerequisites logical, progression clear)

Stage 4: QA & Compliance

Mandatory convergence point - all paths meet here

Project Memory

Custom Instructions

Project Description

Core job: Validate all units meet compliance, quality, and framework requirements before finalisation

What comes in:

  • Complete drafted units (from External Drafting Stage 3A or Internal Drafting Stage 3B)
  • Change registers (if from Internal Drafting - comprehensive modification tracking)
  • Self-QA documentation (initial quality checking performed)
  • Framework mapping documentation (transparent audit trails)
  • External writer self-QA (if from External Drafting path)

What goes out:

  • Validated units (all pass compliance verification):
    • TPOF compliance confirmed
    • AQF alignment verified
    • Framework/standards alignment confirmed
    • Assessment validity verified
    • Internal consistency confirmed
    • Cross-unit consistency verified
  • Comprehensive change registers (complete modification history, rationale clear, impact noted, version control, audit trail)
  • Compliance reports:
    • TPOF compliance report with evidence
    • AQF alignment report with verification
    • Framework alignment report (DigComp, ASFI, sector-specific) with mapping
    • Complete coverage verified (no gaps, no unintended duplication)
  • Alignment matrices:
    • Framework mapping tables (each unit mapped to specific framework elements)
    • Coding system transparent (K1-K13, S1-S6, PE1-PE4 audit trail)
    • Proficiency indicators aligned
    • Coverage complete and systematic
  • Cross-unit consistency tables (terminology, prerequisites, assessment language, Foundation Skills)
  • Technical Committee submission package:
    • Validated units ready for review
    • Change registers (showing iterative refinement)
    • Compliance reports (demonstrating readiness)
    • Alignment matrices (proving framework fidelity)
    • Status report (what's complete, in progress, issues encountered/resolved, decisions needed)
  • Validation records:
    • Technical expert validation documentation
    • Industry SME validation records
    • Cultural validation documentation (First Nations co-design for Sustainable Finance)
    • External validation records (consultant feedback if applicable)
  • Readiness confirmation:
    • For Technical Committee: units, documentation, presentation materials ready
    • For DEWR: milestone requirements met, compliance demonstrated
    • For public consultation: quality sufficient for stakeholder review

Breaks downstream if:

  • Framework misalignment not caught (units don't reflect source framework accurately)
  • AQF inappropriateness not corrected (language says "AQF 4" but complexity is AQF 3)
  • Assessment invalidity not fixed (Performance Evidence not observable, Conditions unrealistic)
  • Inconsistencies not resolved (terminology varies, prerequisites illogical)
  • Regulatory non-compliance not addressed (ACMA rules wrong, standards outdated)
  • Issues discovered too late for efficient correction (fundamental problems at final review requiring major rework)
  • Pattern issues not identified (same problem repeating across multiple units)
  • Vertical progression problems undetected (gaps or jumps in capability development)
  • Cross-sector applicability not verified (examples too narrow, contextualisation insufficient)
